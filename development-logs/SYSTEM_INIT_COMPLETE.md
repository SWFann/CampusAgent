## 🔄 工作流程

### 开始新任务

```bash
# 1. 复制模板
cp development-logs/templates/task-template.md \
   development-logs/in-progress/P0-XX-xxx.md

# 2. 填写任务基本信息
# 3. 开始开发
```

### 完成任务

```bash
# 1. 更新任务日志（验收标准、测试结果等）
# 2. 移动到完成目录
mv development-logs/in-progress/P0-XX-xxx.md \
   development-logs/completed/

# 3. 更新 PROGRESS.md
# 4. 提交代码
git add .
git commit -m "feat(p0): complete P0-XX task description"
```

### 记录阻塞

```bash
# 在任务日志中记录阻塞原因
# 更新 PROGRESS.md 的阻塞项表格
```

## 📝 模板说明

任务日志模板包含以下章节：

1. **Frontmatter** - 元数据（任务ID、状态、时间、耗时）
2. **目标** - 任务描述和验收标准
3. **实现过程** - 详细记录开发步骤
4. **修改的文件** - 新增/修改/删除文件清单
5. **测试结果** - 各类测试通过情况
6. **问题与解决** - 遇到的问题和解决方案
7. **下一步** - 依赖任务和注意事项
8. **提交信息** - Commit、PR、评审人

## 🎓 使用原则

1. **边做边记**：不要事后补记，实时更新
2. **详细记录**：记录决策、问题、解决方案
3. **诚实透明**：如实记录耗时、问题和阻塞
4. **及时更新**：完成任务后立即更新状态
5. **团队可见**：所有日志对团队成员开放
