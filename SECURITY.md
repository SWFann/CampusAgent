# 安全与隐私策略

## 报告问题

请不要在公开 Issue 中披露可利用的安全漏洞、真实用户数据或密钥。仓库维护者应在 GitHub 启用 Private Vulnerability Reporting；启用前，请通过项目所有者指定的私密渠道报告。

报告应尽量包含影响范围、复现条件、受影响版本和建议修复方式，但不得附带不必要的真实个人数据。

## 安全底线

- 不提交 `.env`、访问令牌、数据库快照、私钥或真实模型凭据；
- P2/P3/P4 数据不得出现在普通消息、日志、指标、Trace 或错误详情中；
- 私有场景数据必须加密、按目的授权、带 TTL，并可审计和删除；
- 管理员默认无权读取个人记忆、原始偏好或心理健康数据；
- 模型调用不保存思维链，不默认把敏感内容发往外部供应商；
- 授权、加密或隔离失败时拒绝处理；
- 心理支持能力默认关闭，不能用于自动诊断、隐蔽监控或纪律评价。

数据分类和保留策略见 [隐私工程基线](docs/privacy/PRIVACY_BASELINE.md)。
