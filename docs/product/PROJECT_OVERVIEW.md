# 项目概览

## 定位

CampusAgent 是校园通讯、组织协作和个人智能体的统一底座。个人智能体可以在用户明确授权后参与低风险协商，但不能替代用户作出高风险决定，也不能成为学校读取学生私有状态的入口。

## 核心用户

- 学生：管理个人智能体、记忆和授权，参与通讯、组织与场景；
- 教师：创建课程组织和讨论空间；
- 组织管理员：管理指定班级、宿舍或社团，不读取成员私有偏好；
- 校方管理员：管理账号、组织、模型和节点，仅查看脱敏运行指标；
- 系统管理员：负责配置、运维和安全审计，仍受数据目的和权限约束。

全局角色和组织内角色分离。学生可以同时是全局 `STUDENT`、社团 `OWNER` 和课程 `MEMBER`。

## 六项平台能力

1. 校园身份与人员目录；
2. 学院、班级、宿舍、社团和课程组织；
3. 私聊、群聊、课堂频道和场景会话；
4. 个人、组织和协调智能体；
5. 私有记忆、授权、审计和数据生命周期；
6. 模型、边缘节点和场景插件管理。

## MVP 成功标准

- 新用户注册后自动拥有个人智能体；
- 四名学生可以组成宿舍组织和群聊；
- 每人只在私有界面提交聚餐偏好；
- 群聊看不到个人预算、自由文本或智能体辩论；
- 系统输出三项候选、聚合理由、投票和确认结果；
- 用户可以查看记忆、授权和访问记录；
- 场景结束后临时数据被删除；
- 管理后台展示模型与节点状态，但没有私有偏好读取入口。

## 关键风险控制

- 范围过大：只完整实现聚餐场景；
- 模型不稳定：结构化输出、固定状态机、有限评分、规则/Mock 降级；
- 隐私流于口号：独立私有表、加密、TTL、审计和越权测试；
- 并行冲突：先冻结契约，模块只通过公开 Service 和领域事件协作；
- 边缘节点失败：统一模型网关、健康检查和离线备用路径。
