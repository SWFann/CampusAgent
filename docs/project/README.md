# 项目状态文档

本目录只存放阶段状态、审计结论和整改计划。它回答的问题是：项目当前处在哪个阶段，哪些门禁已经通过，哪些事项仍然阻塞。

## 当前权威顺序

1. [P0/P1 整改计划](P0_P1_REMEDIATION_PLAN.md)：任务清单和整改顺序。
2. [P0 复审记录](P0_REVIEW_RECORD.md)：P0 契约和隐私设计的审计依据。
3. [P1 完成总结](P1_COMPLETION_SUMMARY.md)：工程底座的完成情况和剩余风险。
4. [P0 完成总结](P0_COMPLETION_SUMMARY.md)：历史总结，仅在与复审记录一致时作为参考。

## 使用规则

- Claude 或 Codex 执行任务时，以整改计划中的任务编号为最小工作单元。
- 每个任务完成后，必须有验证结果和开发日志证据。
- 如果完成总结与复审记录冲突，优先相信复审记录。
- 不要在本目录放实现细节、临时代码、命令输出或长篇过程记录；这些内容放入 `development-logs/`。
