# ADR-004：模型路由策略

**状态**：Accepted  
**日期**：2026-07-14  
**决策者**：开发团队

## 背景

需要统一管理模型调用，确保隐私和安全。

## 核心原则

1. **统一入口**：所有模型调用必须经过 Model Gateway
2. **隐私优先**：敏感数据优先本地节点
3. **默认禁用外发**：外部模型默认关闭
4. **降级可用**：节点失败时降级到 Mock/规则

---

## 路由策略

```
请求 → Model Gateway → 路由决策
  ├─ 检查隐私上下文
  ├─ 检查节点健康
  ├─ 检查外部模型是否启用
  └─ 选择路由
```

### 路由优先级

1. **本地边缘节点**（敏感数据时优先）
2. **外部模型 API**（仅授权且启用时）
3. **Mock 模型**（备用）
4. **规则引擎**（最终降级）

---

## 隐私上下文

```json
{
  "contains_sensitive_data": true,
  "allow_external_provider": false,
  "purpose": "meal_planning"
}
```

**规则**：
- `contains_sensitive_data = true` → 禁止路由到外部
- `allow_external_provider = false` → 外部模型不可用

---

## 降级策略

| 场景 | 降级方案 |
|------|---------|
| 本地节点失败 | Mock 模型 |
| Mock 失败 | 规则引擎 |
| 所有路径失败 | 返回错误，不降级为公开 |

**关键**：隐私能力不可降级，只能降级为 Mock/规则。

---

## 元数据记录

**只记录**：
- 调用ID
- 模型名
- Token 数
- 延迟
- 状态码
- 输入/输出哈希

**不记录**：
- ❌ 原始 Prompt
- ❌ 完整响应
- ❌ 思维链

---

## 后果

- 统一的模型管理
- 隐私保护可审计
- 演示可用性高（Mock/规则）
- 外部数据泄露风险低
