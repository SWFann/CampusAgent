# ADR-005：数据保留与清理策略

**状态**：Accepted  
**日期**：2026-07-14  
**决策者**：开发团队

## 背景

需要明确各类数据的保留期限和清理策略，确保隐私合规。

## 数据保留策略

| 数据类型 | 保留期限 | 清理方式 | 原因 |
|---------|---------|---------|------|
| 用户账号 | 永久 | 软删除 | 业务需要 |
| 组织 | 永久 | 归档 | 历史记录 |
| 消息 | 永久 | 软删除 | 用户控制 |
| **私有场景提交** | **立即删除** | **定时任务** | **P4 临时数据** |
| **偏好胶囊** | **立即删除** | **定时任务** | **P4 临时数据** |
| **私有候选评价** | **立即删除** | **定时任务** | **P4 临时数据** |
| 场景结果 | 永久 | - | 业务需要 |
| Agent Run | 30天 | 定时任务 | 元数据 |
| 审计日志 | 90天 | 定时任务 | 合规要求 |
| 长期记忆 | 用户控制 | 用户删除 | 用户主动确认 |

---

## 关键决策

### 1. 临时数据立即清理

**决定**：场景结束后立即删除 P4 数据

**原因**：
- P4 是最高敏感等级
- 最小化数据暴露窗口
- 24 小时兜底防止清理失败

**实现**：
- 场景完成后立即触发清理
- 异步清理任务
- TTL 兜底（expires_at ≤ 24h）
- 定期扫描过期数据

---

### 2. Agent Run 不保存原始内容

**决定**：只保存哈希，不保存原始 Prompt/响应

**原因**：
- 防止敏感内容泄露
- 哈希用于验证完整性

---

### 3. 审计日志 90 天

**决定**：审计日志保留 90 天后自动删除

**原因**：
- 满足合规要求
- 避免无限积累
- 90 天足够追踪问题

---

### 4. 长期记忆用户控制

**决定**：记忆由用户主动确认后保留，可随时删除

**原因**：
- 用户拥有自己的数据
- 隐私优先原则

---

## 清理机制

### 定时任务

```python
# 每5分钟运行一次
async def cleanup_expired_data():
    # 1. 清理过期的私有提交
    await cleanup_private_submissions()

    # 2. 清理过期的胶囊
    await cleanup_capsules()

    # 3. 清理过期的私有评价
    await cleanup_evaluations()

    # 4. 清理过期的 Agent Run
    await cleanup_agent_runs()

    # 5. 清理过期的审计日志
    await cleanup_audit_logs()
```

### 验证

- P8 后必须跑清理测试
- P12 前全量测试
- 发布前验证清理成功

## 后果

- 隐私合规可审计
- 临时数据风险最小化
- 用户数据自主可控
