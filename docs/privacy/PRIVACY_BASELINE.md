# 隐私工程基线

## 数据分类

| 等级 | 示例 | 基线要求 |
|---|---|---|
| P0 公开 | 公开组织名称 | 普通存储 |
| P1 内部 | 班级成员关系 | 鉴权与可见性控制 |
| P2 私有 | 饮食偏好、预算 | 应用层加密、逐目的授权、审计 |
| P3 高敏感 | 心理状态、咨询记录 | 独立域、强隔离、最小访问 |
| P4 临时秘密 | 原始场景输入、私有评价 | 临时加密存储、TTL、自动销毁 |

## 授权模型

敏感访问必须同时明确：`who`、`what`、`purpose`、`scope`、`expiration` 和 `consent`。撤销后新请求立即失效；不能用一次全局授权覆盖所有未来场景。

个人智能体默认 L1（建议），聚餐场景最高 L2（受限代言）。最终投票确认、支付、报名、对外分享和心理支持联络必须由用户本人确认。

## 聚餐数据流

1. 原始偏好只进入当前用户的私有提交域；
2. 个人智能体抽取最小化结构化胶囊；
3. 协调层以胶囊评价候选，只返回分数、硬约束结果和安全理由码；
4. 公共结果只包含候选排名、聚合理由和隐私说明；
5. 场景结束后删除原始输入、胶囊、私有评价和模型中间响应；
6. 写入长期记忆必须另行确认。

## 保留策略

| 数据 | 默认期限 |
|---|---|
| 原始私有场景提交 | 场景结束立即删除，最长 24 小时 |
| 偏好胶囊 | 场景结束删除，最长 24 小时 |
| 私有候选评价 | 场景结束删除，最长 24 小时 |
| 最终场景结果 | 可保留 |
| Agent Run 元数据 | 30 天 |
| 系统审计元数据 | 90 天 |
| 长期记忆 | 用户主动确认后保留，可随时删除 |

## 日志与模型

- `LOG_PROMPT_CONTENT=false` 必须是生产默认值；
- 不记录原始偏好、完整 Prompt、完整模型响应或思维链；
- 仅记录调用 ID、逻辑模型、Token、延迟、状态、脱敏错误和结构化结果哈希；
- 外部模型默认关闭；敏感数据仅在明确授权且策略允许时才能外发；
- 模型不可用可降级到规则或 Mock，但隐私能力不可降级。

## 必测隐私断言

- A 无法读取 B 的偏好；
- 群主和 SchoolAdmin 均无法读取 P2/P3 正文；
- 普通聊天接口查不到私有提交；
- 插件不能绕过 Memory Service；
- 日志和指标不含敏感正文；
- 场景结束后临时数据确实删除；
- 取消授权后新访问被拒绝；
- 数据导出只包含当前用户数据；
- 隐私依赖故障时场景失败关闭。
