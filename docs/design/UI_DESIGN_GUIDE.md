# CampusAgent UI 初步设计与自然语言设计规范

> 版本：v0.1
>
> 日期：2026-07-14
>
> 目标：为比赛 MVP 建立统一、现代、可信且可实现的 Web UI 方向

## 1. 设计定位

CampusAgent 的界面不是“另一个 AI 聊天框”，而是一套校园通讯与智能体协作工作台。视觉上应让用户首先感到熟悉、安静和可控，其次才感受到 AI 的存在。

建议设计主题：**Quiet Intelligence / 安静的智能**。

界面借鉴以下成熟产品的设计原则，但不直接复制其品牌资产、布局细节或视觉标识：

- QQ、微信：熟悉的会话列表—聊天内容—会话信息结构，低学习成本；
- X：清晰的内容流、紧凑的信息层级和即时操作反馈；
- Apple：克制留白、精确排版、半透明层次、连续动效与强一致性；
- 企业协作产品：可扩展的组织、成员、权限和管理后台结构。

整体气质应当是：**清爽、可信、年轻、克制、带一点未来感，但不做霓虹赛博风。**

## 2. 核心视觉原则

### 2.1 通讯优先，AI 后置

用户进入产品后看到的是校园关系、最近消息和当前任务。智能体作为可见但不喧宾夺主的协作者出现，不要在每个页面放巨大的 AI 输入框或渐变光效。

### 2.2 隐私状态必须可见

任何私有输入页面都必须在提交前说明：谁能看到、用于什么、保存多久、是否写入长期记忆。隐私信息不是藏在设置里的法律文本，而是任务流程的一部分。

### 2.3 公开与私有区域有稳定差异

- 普通聊天：中性画布和常规消息气泡；
- 私有智能体区域：轻微的主色调背景或细边框，并出现锁形图标；
- 场景公共结果：白色/中性卡片和明确的“聚合结果”标签；
- 高风险或不可逆操作：独立确认对话框，不使用普通主按钮直接完成。

差异不能只依赖颜色，必须同时使用图标、标题和说明文字。

### 2.4 信息密度分层

消息列表和组织目录保持类似微信/QQ 的紧凑密度；场景表单、记忆管理和授权页面使用 Apple 式更宽松的留白；管理后台使用清晰表格和指标，不堆叠装饰性卡片。

## 3. 品牌与设计 Token

### 3.1 色彩

建议主色为 **Campus Indigo**，表达可信、智能和年轻感；隐私成功态使用独立的绿色语义色。

| Token | 浅色模式 | 深色模式 | 用途 |
|---|---|---|---|
| Canvas | `#F5F7FA` | `#0E1014` | 应用背景 |
| Surface | `#FFFFFF` | `#171A20` | 面板、弹层、卡片 |
| Surface Subtle | `#EEF1F6` | `#20242C` | hover、选中前背景 |
| Text Primary | `#17191D` | `#F5F7FA` | 标题、正文 |
| Text Secondary | `#667085` | `#A7AFBE` | 辅助说明、时间 |
| Border | `#E2E6EC` | `#2B303A` | 分隔线、输入框 |
| Campus Indigo | `#5965E8` | `#7B84FF` | 主操作、选中态 |
| Indigo Soft | `#EEF0FF` | `#262A4A` | 私有智能体轻背景 |
| Privacy Green | `#168A5B` | `#43C98B` | 已保护、已清理、有效授权 |
| Warning | `#B66A05` | `#F2B84B` | 即将过期、需注意 |
| Danger | `#C93A3A` | `#FF6B6B` | 撤销、删除、错误 |

使用规则：主色只用于当前导航、主按钮、焦点和关键进度；大面积背景保持中性；渐变只允许出现在品牌插画或空状态，不用于普通按钮、表单和大面积面板。

### 3.2 字体与字号

字体栈：`Inter, SF Pro Display, PingFang SC, Microsoft YaHei, sans-serif`。

| 层级 | 字号/行高 | 字重 | 用途 |
|---|---|---|---|
| Display | 28/36 | 600 | 登录欢迎语、少量页面首屏 |
| H1 | 24/32 | 600 | 页面标题 |
| H2 | 18/26 | 600 | 区块标题、会话名称 |
| Body | 14/22 | 400 | 默认正文 |
| Label | 13/18 | 500 | 字段标签、导航、按钮 |
| Caption | 12/18 | 400 | 时间、来源、帮助信息 |

中文正文不要低于 13px。正文不使用超细字重；长段落最大宽度约 680px。

### 3.3 圆角、阴影和间距

- 基础间距使用 4px 网格：4、8、12、16、20、24、32、40；
- 输入框、按钮圆角 10px；卡片 14px；弹层 16px；头像为圆形；
- 面板主要依靠 1px 分隔线建立层级，阴影保持非常轻；
- 浮层阴影使用柔和大扩散，不使用纯黑硬阴影；
- 页面主内容左右留白：桌面 24–32px，平板 20px，手机 16px。

## 4. 桌面端信息架构

推荐采用四区布局：

```text
┌────────┬──────────────┬──────────────────────────┬──────────────┐
│ 全局栏 │ 列表/目录区  │ 主内容区                 │ 上下文侧栏   │
│ 72 px  │ 280–320 px   │ min 520 px / 自适应      │ 280–320 px   │
└────────┴──────────────┴──────────────────────────┴──────────────┘
```

### 全局导航栏

深浅模式均使用中性表面，不做高饱和整栏。顶部为 CampusAgent 标志；中间依次为首页、消息、联系人、组织、智能体、场景、记忆；底部为头像、设置和管理员入口。当前项使用柔和主色底、主色图标和左侧/内部选中指示，不只改变图标颜色。

### 列表/目录区

消息页展示搜索、筛选和会话列表；联系人页展示校园组织树；场景页展示场景类别。列表行高 64–72px，包含头像、标题、摘要、时间和未读数。当前行使用轻微选中背景，不使用厚重边框。

### 主内容区

消息页由顶部会话栏、可滚动内容流和底部输入区组成。场景卡嵌入内容流，但尺寸和层级高于普通消息；私有偏好表单不得嵌在普通消息输入框中，点击场景卡后进入独立私有页面或侧滑层。

### 上下文侧栏

根据当前页面显示群成员、群智能体、当前场景、共享文件、授权摘要或候选详情。侧栏可关闭，在 1280px 以下默认收起为抽屉。

## 5. 核心页面设计

### 5.1 首页

首页顶部使用一句简短问候和隐私状态摘要，不做巨型 Banner。下方按优先级排列：待处理授权/邀请、进行中的场景、最近消息、智能体状态。最多展示三类核心内容，避免“数据驾驶舱式”卡片堆叠。

### 5.2 消息页

这是最重要的页面。视觉结构接近用户熟悉的桌面通讯软件：左侧会话列表、中央聊天、右侧上下文。本人消息使用主色浅背景，其他人消息使用中性表面；系统消息居中且弱化；智能体消息必须标识智能体身份与代表对象。

顶部会话栏显示群名、人数、当前场景状态和通话/详情等次要操作。聊天输入区保持简洁，文件、表情和场景入口放在同一工具行。

### 5.3 聚餐场景卡

场景卡包含标题、当前阶段、参与进度、隐私说明和一个主操作。等待输入时突出 `3/4 已提交`，但不显示谁提交了什么；候选完成后展示前三项及匹配度；投票结束后只显示聚合理由和最终结果。

卡片固定包含一行隐私说明：`个人偏好仅用于本次协商，其他成员与管理员不可见。`

### 5.4 私有偏好页

页面顶部首先显示锁形图标、用途、访问者、保存期限和是否会写入长期记忆。表单分为“必须满足”和“偏好倾向”两组，预算、距离、时间采用结构化控件，自由文本位于最后并标注不会公开。

底部主按钮使用“私密提交偏好”，旁边显示自动清理时间。不要使用“发送”这个容易与群消息混淆的词。

### 5.5 智能体中心

顶部展示智能体头像、名称、当前代理等级和“由你拥有”的说明。主体分为对话、表达风格、场景权限、记忆使用和最近运行。代理等级必须用分段控制和自然语言解释，不能只显示 L0/L1/L2。

### 5.6 记忆中心

以可筛选列表展示记忆内容、分类、敏感等级、来源、最近使用、过期时间。敏感正文默认折叠，用户主动展开；删除、撤销共享和导出使用明确操作。右侧或详情抽屉展示访问时间线，不出现模型思维链。

### 5.7 管理后台

使用克制的表格、趋势图和状态列表展示用户数、组织数、模型请求、节点健康、延迟和错误率。页面顶部明确写明：`管理员无法查看个人偏好、私有记忆和协商过程。` 不得设计任何暗示可以进入用户私有数据的入口。

## 6. 关键组件规范

| 组件 | 必须包含 | 不允许 |
|---|---|---|
| PrivacyNotice | 锁图标、用途、可见范围、保存期限 | 只写“我们重视隐私”的空泛文案 |
| ConsentCard | 场景、代理能力、记忆类别、到期时间、允许/拒绝 | 默认勾选永久授权 |
| SceneProgress | 阶段名称、提交人数、截止时间、退出操作 | 展示成员的具体偏好 |
| CandidateCard | 名称、匹配度、预算/距离公共属性、聚合理由、投票 | 指出某个成员导致的限制 |
| AgentBadge | 智能体类型、代表对象 | 假装为真实用户 |
| MemoryRow | 类别、来源、敏感级别、更新时间、使用记录 | 默认展示全部敏感正文 |
| AuditRow | 时间、动作、资源类型、目的、结果 | Prompt、偏好正文、模型完整响应 |
| EmptyState | 清楚说明为什么为空和下一步动作 | 无意义插画占满页面 |

## 7. 交互与动效

- 页面切换 160–220ms，使用淡入与 4–8px 位移；
- 侧栏/抽屉 220–280ms，使用自然缓动；
- 新消息只轻微上移出现，不弹跳、不闪烁；
- 场景阶段变化用状态文字、进度和轻微背景过渡表达；
- “私密提交”成功后显示锁闭合、结构化胶囊已生成、原文清理时间；
- 删除记忆、撤销授权、确认最终方案必须二次确认；
- 支持 `prefers-reduced-motion`，关闭非必要动画。

## 8. 响应式策略

- ≥1440px：四区完整布局；
- 1024–1439px：全局栏 + 列表 + 主内容，上下文为抽屉；
- 768–1023px：窄全局栏 + 主内容，列表和上下文使用抽屉；
- <768px：底部导航；消息页在“会话列表”和“聊天详情”之间单页切换；
- 手机私有表单使用单列，主提交按钮在内容底部，不遮挡最后一个字段。

比赛首发仍以桌面 Web 为主，但组件从一开始就使用 Auto Layout/响应式约束，避免后期重做。

## 9. 无障碍与可用性

- 正文与背景对比度至少符合 WCAG AA；
- 所有图标按钮有可读标签，状态不只靠颜色；
- 键盘可完成登录、搜索、聊天、提交偏好、投票和撤销授权；
- 焦点环清晰，不移除浏览器焦点样式；
- 错误信息紧邻字段，并说明如何修复；
- 重要隐私说明使用普通可读文本，不放在 Tooltip 里；
- 时间、金额和距离使用本地化格式。

## 10. 给 UI 设计师的执行顺序

1. 在 Figma 建立颜色、排版、间距、圆角和阴影 Variables；
2. 先设计导航、按钮、输入框、头像、列表行、对话气泡和弹层；
3. 完成消息页桌面骨架，这是全产品的视觉基准；
4. 设计 SceneProgress、PrivacyNotice、ConsentCard、CandidateCard；
5. 串联“群聊发起 → 私有提交 → 候选 → 投票 → 确认 → 清理”的完整原型；
6. 再扩展首页、组织、智能体、记忆和管理后台；
7. 补齐浅色/深色、空/加载/错误/无权/过期/离线状态；
8. 做 1440、1024、768、390 四个断点检查；
9. 让未参与项目的人完成一次可用性走查；
10. 最后冻结组件命名、Token 和开发标注。

推荐 Figma 页面：`00 Foundations`、`01 Components`、`02 Auth`、`03 Home`、`04 Messages`、`05 Scene Meal`、`06 Agent`、`07 Memory`、`08 Admin`、`09 Prototype`。

## 11. 可直接交给 UI 设计工具的自然语言描述

设计一套名为 CampusAgent 的现代校园智能体协作平台 Web UI。产品服务于高校学生，核心是校园通讯、组织关系、个人智能体、私有记忆和多智能体场景协商。整体风格为“安静的智能”：清爽、可信、年轻、克制，带轻微未来感，但不要霓虹、复杂玻璃拟态或夸张渐变。

界面借鉴微信和 QQ 桌面端熟悉的会话列表与聊天布局、X 清晰紧凑的内容流、Apple 的留白、排版、材质层次和连续动效，但不要复制任何品牌 Logo、图标、配色或像素级布局。

桌面端使用四区结构：72px 全局导航栏、280–320px 会话或目录列表、自适应主内容区、280–320px 上下文侧栏。主色使用克制的靛蓝色，背景和面板使用低对比中性色，隐私保护使用语义绿色。圆角中等，边框细，阴影轻，排版使用 Inter、SF Pro 和苹方。支持浅色与深色模式。

重点设计消息页：左侧显示校园群聊和私聊，中间显示宿舍群聊天，右侧显示群成员、群智能体和当前场景。聊天流中嵌入“宿舍聚餐去哪”场景卡，显示 3/4 人已私密提交、当前阶段、截止时间和隐私说明，但绝不展示任何人的预算、禁忌或自由文本。

设计独立的私有偏好页面，顶部明确显示锁图标、用途、可见范围和最长 24 小时自动清理。表单包括预算上限、菜系、饮食禁忌、距离、时间、环境和其他说明，底部按钮写“私密提交偏好”，不能写成普通聊天的“发送”。

候选结果卡展示三家餐厅、匹配度、公共预算区间、距离和聚合理由，例如“满足全部成员的硬性限制”，不能指出具体成员。投票后由发起人进行人工确认，随后显示临时数据已清理，并询问每个用户是否愿意把结果写入自己的长期记忆，默认不保存。

同时设计首页、联系人和组织、个人智能体中心、记忆中心、场景中心和管理员后台。管理员后台只展示系统指标、模型和边缘节点状态，并明确标注管理员无法查看个人偏好、私有记忆和协商过程。所有页面必须包含加载、空、错误、无权限、授权过期和离线状态，并满足键盘操作、清晰焦点和 WCAG AA 对比度。

## 12. 设计评审检查表

- [ ] 第一眼能看出是校园通讯与协作平台，而非通用 AI Chat；
- [ ] 用户能区分普通群聊、私有智能体和公开场景结果；
- [ ] 提交偏好前能回答“谁看、为何用、存多久”；
- [ ] 群主、其他成员和管理员界面都没有私有偏好入口；
- [ ] 主流程只有一个明确主操作，危险操作有二次确认；
- [ ] 未完成场景被标注为概念或即将上线；
- [ ] 浅色、深色和四个响应式断点均可用；
- [ ] 状态不只通过颜色表达；
- [ ] 中文正文清晰、无过小字号或过低对比度；
- [ ] 组件和 Token 可直接映射到前端设计系统。
